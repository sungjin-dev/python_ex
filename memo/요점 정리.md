1. round(total, 0)은 결과가 '실수(.0)'로 나옵니다자릿수를 0으로 지정하면 소수점 아래를 다 날리기는 하지만, 데이터 타입은 여전히 실수형(float)을 유지합니다.round(3.5, 0) $ -> $ 4가 아니라 4.0round(3.1, 0) $ -> $ 3가 아니라 3.0만약 정수형(int)인 4나 3으로 완벽하게 바꾸고 싶다면 round(total, 0) 대신 아예 타입을 바꾸는 int(total)을 쓰는 게 좋습니다. (단, int()는 반올림이 아니라 소수점을 그냥 버려버립니다!)

# .copy() 얕은 복사 -> 알맹이 상자들은 원래 있던 것을 그대로 재사용


list_a = [[1, 2], [3, 4]]는 "큰 상자 안에 [1, 2] 상자와 [3, 4] 상자, 총 2개의 작은 상자가 들어있는 구조"

1. 얕은 복사 (list_b = list_a.copy())를 했을 때

   1) 파이썬은 list_b라는 새로운 큰 상자 생성
   2) 그 속에 알맹이를 기존 list_a 안에 있던 [1, 2] 상자와 [3, 4] 상자를 그대로 선으로 연결(참조)만 함
   3) 결과적으로 큰 상자(list_a, list_b)는 주소가 다르지만, 그 안의 작은 상자들은 같은 주소를 공유

 -> 그래서 list_b[0][0] = 99로 첫 번째 작은 상자의 알맹이를 바꾸면, list_a도 똑같은 작은 상자를 바라보고 있기 때문에 둘 다 값이 변해버리는 것입니다.

2. 깊은 복사 (deepcopy)가 필요한 이유★
 "완벽하게 독립된 복사본"을 만들려면 큰 상자뿐만 아니라 그 안에 들어있는 작은 상자들까지 전부 새것으로 복제

 import copy

list_a = [[1, 2], [3, 4]]
# 깊은 복사 실행! (작은 상자까지 전부 새로 만듦)
list_b = copy.deepcopy(list_a)

list_b[0][0] = 99
print(list_a)  # 출력: [[1, 2], [3, 4]] -> list_a는 안전하게 유지됨!
print(list_b)  # 출력: [[99, 2], [3, 4]]  -> list_b만 변경



다차원 리스트 list_a = [[1, 2], [3, 4]]를 list_b = list_a.copy()로 복사했을 때 발생하는 현상은?


list_b의 내부요소를 수정하면 list_a의 내부요소도 같이 변한다. 


얕은 복사를 하면 중첩된 객체(내부리스트)는 같은 메모리 주소를 가리키기 때문에 한쪽을 수정하면 같이 바뀜


깊은 복사가 필요한 이유는 리스트 안의 리스트가 들어있는 다차원 구조를 복사할 떄이다.

📂 1. 폴더 이동 및 확인 

cd(Change Directory)

cd .. : (핵심) 현재 위치에서 한 단계 상위 폴더로 빠져나갑니다. (cd ../..를 치면 두 단계 위로 갑니다.)

dir  : 현재 폴더 안에 어떤 파일과 폴더들이 있는지 목록을 보여줍니다. 

cls  : 터미널 화면에 글자가 너무 많아 복잡할 때, 화면을 깨끗하게 지워줍니다. (실제 파일이 지워지는 건 아닙니다!)

mkdir 폴더명 : 새 폴더를 만듭니다. (예: mkdir test_folder)

-m 옵션은 모듈(module)을 실행

python -m pip install 패키지명

외부 라이브러리(예: requests, pandas)를 설치할 때

python -m pip list

내 컴퓨터(또는 현재 환경)에 설치된 파이썬 패키지 목록과 버전을 쫙 보여줍니다.

python -m pip freeze > requirements.txt

현재 프로젝트에 설치된 패키지들의 목록을 requirements.txt라는 텍스트 파일로 저장해 줍니다. 나중에 다른 컴퓨터나 학원에서 이 파일만 있으면 똑같은 환경을 1초 만에 세팅할 수 있습니다.

🛡️ 3. 가상환경 (프로젝트마다 방 따로 쓰기)

python -m venv venv

현재 폴더 안에 venv라는 이름의 독립적인 파이썬 가상환경(작은 방)을 만듭니다.

.\venv\Scripts\activate

만들어둔 가상환경 방으로 입장(실행)합니다. 성공하면 터미널 맨 앞에 (venv)라는 초록색 글씨가 생깁니다.

deactivate

가상환경 방에서 다시 원래 컴퓨터 환경으로 빠져나옵니다.
